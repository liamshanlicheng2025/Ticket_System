# 火车票订票系统 (Ticket System)

一个类似 12306 的火车票订票系统后端实现，支持用户管理、车次管理、余票查询、购票、候补购票、订单管理等功能。  
本项目为某高校编程大作业，完全使用 C++ 实现，**不依赖任何 STL 容器或算法**（除 `std::string` 外），所有数据持久化存储在本地文件中，并通过自研 B+ 树索引进行高效检索。

---

## ✨ 主要功能

| 类别 | 功能 | 指令示例 |
|------|------|----------|
| **用户管理** | 注册、登录、登出、查询/修改个人信息 | `add_user`, `login`, `logout`, `query_profile`, `modify_profile` |
| **车次管理** | 添加、删除、发布车次，查询车次详情 | `add_train`, `delete_train`, `release_train`, `query_train` |
| **车票查询** | 查询直达车次（支持按时间/价格排序） | `query_ticket` |
| **换乘查询** | 查询一次换乘的最优方案（时间/价格最优） | `query_transfer` |
| **购票与候补** | 购买车票，余票不足时可加入候补队列 | `buy_ticket` |
| **订单管理** | 查看历史订单（按时间倒序），退票 | `query_order`, `refund_ticket` |
| **系统维护** | 清除所有数据，退出程序 | `clean`, `exit` |

---

## 🚀 技术特性

- **B+ 树索引**：自实现 B+ 树（`BPTree`），支持插入、删除、精确查找、**前缀扫描**，所有索引数据持久化到磁盘。
- **多级缓存**：LRU 缓存池（`CacheSlot`），缓存最近访问的 B+ 树节点，减少磁盘 I/O。
- **数据与索引分离**：每条业务记录存储在独立的二进制文件中（如 `users.dat`），B+ 树仅存储 `(key, record_id)` 映射，实现高效定位。
- **零 STL 容器**：除 `std::string` 外，全部使用原生数组和指针，满足严格内存限制。
- **高性能 I/O**：使用 C 标准库文件操作（`fopen`/`fread`/`fwrite`）和自定义快速输入输出（FastInput/FastOutput 可选），适用于百万级指令压力。
- **UTF-8 中文支持**：车站名、用户姓名等均使用 UTF-8 编码，B+ 树比较基于 `memcmp`，正确支持中文排序和前缀匹配。
